## 1. 概要

### 1.1 コンセプト

**HereShot** は、写真と撮影場所を結び付け、友達が撮った場所に行くことで写真を「解放」できる位置連動型のソーシャルネットワークアプリである。

遠隔からは地図上のピン・場所情報・ぼかしサムネイルのみ閲覧でき、投稿地点から **50m 以内** に入ったときに初めて写真本体が表示される。Instagram のようにフィードを眺めるのではなく、**友達の思い出を目的地にして外に出る** 体験を提供する。

キャッチコピー案: **「ここで撮った。ここで見る。」**

### 1.2 ターゲットユーザー

- 旅行やお出かけの思い出を場所とともに共有したい人
- 友達と「あの場所に行ってみよう」と繋がりたい人
- 地図ベースで写真を探索するのが好きな人

### 1.3 リリース目標

- **初回リリース: iOS アプリ**（App Store）
- 開発環境: Windows（クラウドビルドで iOS 向けビルド）

### 1.4 差別化ポイント

| 要素 | 既存の地図SNS（mapic 等） | HereShot |
|------|--------------------------|----------|
| 遠隔閲覧 | 写真フル表示 | ぼかしサムネ ＋ ピン・場所情報 |
| 現地閲覧 | 特別な体験なし | 50m 以内で写真本体を解放 |
| 投稿 | カメラロール可が多い | アプリ内カメラのみ |
| GPS 誤差 | 救済 UI ありがち | 救済なし（裏技として楽しむ） |
| UI | フィード中心 | マップ中心 ＋ 中央カメラ |

## 2. 機能要件

### 2.1 投稿

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-POST-01 | 写真は **アプリ内カメラのみ** で撮影できる。カメラロールからの選択・アップロードは不可 | 必須 |
| FR-POST-02 | 撮影時に端末 GPS により **撮影地点の座標（緯度・経度）** を記録する | 必須 |
| FR-POST-03 | 投稿前に公開範囲を選択できる（後述 2.3） | 必須 |
| FR-POST-04 | 投稿完了後、ホーム画面の地図上にピンとして表示される | 必須 |
| FR-POST-05 | 投稿時に場所名（逆ジオコーディングまたはユーザー入力）を紐づけられる | 必須 |
| FR-POST-06 | 投稿時にオリジナル画像と **ぼかしサムネイル** をサーバー側で生成・保存する | 必須 |

### 2.2 閲覧（段階的閲覧モデル）

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-VIEW-01 | **他人の投稿（遠隔）**: 地図ピン・場所名・投稿日時・**ぼかしサムネイル** を表示する。写真本体は非表示 | 必須 |
| FR-VIEW-02 | **他人の投稿（現地）**: 投稿地点から **50m 以内** にいる場合、写真本体をフル表示する | 必須 |
| FR-VIEW-03 | **本人の投稿**: 場所に関係なく、常に写真本体をフル表示できる（GPS ゲートの対象外） | 必須 |
| FR-VIEW-04 | 50m 判定はクライアント側で Haversine 公式により行う | 必須 |
| FR-VIEW-05 | GPS 誤判定時の救済 UI（精度表示・リトライ・半径緩和）は **実装しない** | 必須 |
| FR-VIEW-06 | GPS のブレや誤判定は仕様として許容し、ゲーム性・裏技要素として扱う | 必須 |

#### 閲覧フロー

```
遠隔閲覧
  → 地図 / フィードを開く
  → 権限のあるピンを表示
  → 場所名・投稿日時・ぼかしサムネ
  → 本人? → はい: 写真本体フル表示
           → いいえ: ロック状態
  → その場所へ移動
  → 投稿地点から 50m 以内
  → GPS 判定 OK
  → 写真本体フル表示
```

### 2.3 プライバシー・公開範囲

各投稿に以下のいずれかを設定できる。

| 設定値 | 説明 |
|--------|------|
| フォロワーのみ | 投稿者をフォローしているユーザーのみピンを閲覧可能 |
| 相互フォロー | 投稿者と相互フォローのユーザーのみ閲覧可能 |
| 公開 | フォロー関係なく誰でも閲覧可能（地図探索・近傍表示で発見可能） |

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-PRIV-01 | 投稿ごとに上記 3 択から公開範囲を選択できる | 必須 |
| FR-PRIV-02 | 新規投稿のデフォルト公開範囲は **フォロワーのみ** とする | 必須 |
| FR-PRIV-03 | 設定画面でデフォルト公開範囲を変更できる | 推奨 |
| FR-PRIV-04 | ユーザーをブロックできる | 必須 |
| FR-PRIV-05 | ブロックしたユーザー / されたユーザーとの投稿は相互に非表示とする | 必須 |

### 2.4 ソーシャル（フォロー）

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-SOC-01 | ユーザーをフォロー / アンフォローできる | 必須 |
| FR-SOC-02 | ホーム画面の地図には、閲覧権限のある投稿（フォロー関係 × 公開範囲）のピンを表示する | 必須 |
| FR-SOC-03 | 「公開」設定の投稿は、非フォロワーも地図上で発見できる | 必須 |
| FR-SOC-04 | ユーザー名でユーザーを検索できる | 必須 |
| FR-SOC-05 | 他ユーザーのプロフィールと投稿ピン一覧を閲覧できる | 必須 |

### 2.5 地図・場所検索

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-MAP-01 | インタラクティブな地図上に投稿ピンを表示する | 必須 |
| FR-MAP-02 | ユーザーの現在地を地図上に表示する | 必須 |
| FR-MAP-03 | ホーム画面上部に **場所検索窓** を配置する | 必須 |
| FR-MAP-04 | 場所検索により地名・スポット名から座標を取得し、地図をその位置へ移動する | 必須 |
| FR-MAP-05 | 表示範囲内のピンを効率的に取得する（bounding box クエリ） | 必須 |
| FR-MAP-06 | ピンタップで投稿詳細（ぼかしサムネ / 現地解放状態）を表示する | 必須 |

### 2.6 認証・アカウント

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-AUTH-01 | アカウント登録・ログインができる | 必須 |
| FR-AUTH-02 | **Sign in with Apple** に対応する | 必須 |
| FR-AUTH-03 | プロフィール（ユーザー名・アイコン・自己紹介）を編集できる | 必須 |
| FR-AUTH-04 | ログアウトできる | 必須 |

### 2.7 設定

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-SET-01 | アカウント設定（プロフィール編集・ログアウト） | 必須 |
| FR-SET-02 | プライバシー設定（デフォルト公開範囲・ブロック一覧） | 必須 |
| FR-SET-03 | 位置情報の利用に関する説明を表示する | 必須 |
| FR-SET-04 | 利用規約・プライバシーポリシーへのリンク | 必須 |

## 3. 画面要件

### 3.1 ナビゲーション構成

タブバーに 4 タブを配置し、**中央に丸いカメラボタン**（FAB 風）を配置する。

```
┌─────────────────────────────────────┐
│           （各画面コンテンツ）          │
├──────────┬──────────┬──────────────┤
│  ホーム   │ マイページ │              │
│   🗺️    │   👤    │   🔍  ⚙️   │
│          │    (📷)   │  検索  設定   │
└──────────┴──────────┴──────────────┘
```

| タブ | 画面 | 説明 |
|------|------|------|
| ホーム | マップ画面 | 地図・ピン・場所検索 |
| マイページ | マイページ画面 | プロフィール・自分の投稿 |
| （中央） | カメラ | 撮影・投稿フロー（モーダル / フルスクリーン） |
| ユーザー検索 | 検索画面 | ユーザー名検索・フォロー |

### 3.2 ホーム画面（マップ）

| 要素 | 仕様 |
|------|------|

### 3.3 マイページ画面

| 要素 | 仕様 |

| 要素 | 仕様 |

### 3.6 カメラ画面

| 要素 | 仕様 |

## 5. 非機能要件

### 5.1 プラットフォーム

| 項目 | 要件 |
|------|------|
| 初回リリース | iOS（App Store） |
| 最小 iOS バージョン | iOS 16 以上（Expo SDK 準拠、実装時に確定） |
| 将来 | Android 対応を見据えたクロスプラットフォーム構成 |

### 5.2 パフォーマンス

| 項目 | 要件 |
|------|------|
| 地図のピン読み込み | 表示範囲内のピンを 2 秒以内に表示（目標） |
| 写真アップロード | 撮影後 10 秒以内に投稿完了（目標、回線状況による） |

### 5.3 セキュリティ・プライバシー

| 項目 | 要件 |
|------|------|
| 認証 | Supabase Auth + RLS によるアクセス制御 |
| 画像 URL | 閲覧権限に応じてオリジナル / ぼかし URL を出し分け |
| 投稿者の現在地 | 常に非表示（投稿地点のピンのみ表示） |
| 位置情報 | 使用中のみ取得（`When In Use`） |

### 5.4 App Store 申請

| 項目 | 要件 |
|------|------|
| 位置情報 | `NSLocationWhenInUseUsageDescription` |
| カメラ | `NSCameraUsageDescription` |
| Sign in with Apple | 他社ログイン併用時は必須 |
| プライバシーラベル | 位置・写真・ユーザー ID の収集を申告 |

## 6. 技術スタック

### 6.1 推奨構成

| レイヤー | 技術 | 備考 |
|---------|------|------|
| モバイル | Expo (React Native) + TypeScript | Windows 開発可、EAS Build で iOS ビルド |
| ナビゲーション | Expo Router | 4 タブ + カメラモーダル |
| 地図表示 | react-native-maps（iOS: Apple Maps） | 追加 API 料金なし |
| 場所検索 | Mapbox Search API | ホーム上部の検索窓 |
| 位置情報 | expo-location | 50m 判定はクライアント Haversine |
| カメラ | react-native-vision-camera | アプリ内カメラのみ |
| バックエンド | Supabase | Auth / PostgreSQL / Storage / Edge Functions |
| DB 拡張 | PostGIS | マップ bbox クエリ |
| 画像処理 | Supabase Storage + Edge Function | ぼかしサムネ生成 |
| 認証 | Supabase Auth + Sign in with Apple | |
| iOS ビルド | EAS Build | Mac 不要で TestFlight 配信 |
| 状態管理 | TanStack Query + Zustand | |

### 6.2 アーキテクチャ

```
[iOS App - Expo]
  ├─ Expo Router UI
  ├─ react-native-maps（地図・ピン）
  ├─ Mapbox Search（場所検索）
  ├─ Vision Camera（撮影）
  ├─ expo-location → Haversine 50m 判定
  └─ Supabase Client

[Supabase]
  ├─ Auth（Sign in with Apple）
  ├─ PostgreSQL + PostGIS
  │    ├─ profiles
  │    ├─ posts
  │    ├─ follows
  │    └─ blocks
  ├─ Storage（オリジナル + ぼかし画像）
  └─ Edge Function（ぼかしサムネ生成）
```

### 6.3 データモデル（MVP）

#### profiles

| カラム | 型 | 説明 |
|--------|-----|------|
| id | uuid (PK) | auth.users と紐づけ |
| username | text (unique) | ユーザー名 |
| avatar_url | text | アイコン URL |
| bio | text | 自己紹介 |
| default_visibility | enum | デフォルト公開範囲 |
| created_at | timestamptz | 作成日時 |

#### posts

| カラム | 型 | 説明 |
|--------|-----|------|
| id | uuid (PK) | |
| user_id | uuid (FK) | 投稿者 |
| lat | double | 緯度 |
| lng | double | 経度 |
| place_name | text | 場所名 |
| image_url | text | オリジナル画像 URL |
| blurred_url | text | ぼかしサムネ URL |
| visibility | enum | followers / mutual / public |
| created_at | timestamptz | 投稿日時 |

#### follows

| カラム | 型 | 説明 |
|--------|-----|------|
| follower_id | uuid (FK) | フォローする側 |
| following_id | uuid (FK) | フォローされる側 |
| created_at | timestamptz | |

#### blocks

| カラム | 型 | 説明 |
|--------|-----|------|
| blocker_id | uuid (FK) | ブロックする側 |
| blocked_id | uuid (FK) | ブロックされる側 |
| created_at | timestamptz | |

### 6.4 コア機能の実装方針

| 機能 | 実装 |
|------|------|
| 50m 現地解放 | クライアント: `distance(post, user) <= 50` |
| ぼかしサムネ | Edge Function が投稿時に生成。遠隔 API はぼかし URL のみ返す |
| 本人フル表示 | `viewer_id === author_id` ならオリジナル URL を返す |
| プライバシー 3 択 | `posts.visibility` + RLS |
| フォロー制フィード | `follows` JOIN + 可視性フィルタ |
| 場所検索 | Mapbox Search → `animateToRegion` |

## 7. コールドスタート戦略（MVP 後に実装）

MVP 完成後に検証・実装する施策案。

| # | 施策 | 概要 |
|---|------|------|
| 1 | 自分のマップを先に埋める | 初回オンボーディングで 1 枚撮影。一人でも思い出マップとして使える |
| 2 | 公開ピンの近傍表示 | フォロー 0 人でも近くの公開投稿をぼかしサムネで発見 |
| 3 | 招待リンク | 友達招待でお互いのピンが地図に見える |
| 4 | 場所検索 × エリアシード | 特定エリアにベータ投稿を集中 |
| 5 | 解放済みコレクション | 現地で解放した写真をマイページに蓄積 |
| 6 | 裏技文化 | GPS ブレを仕様として発信し、コミュニティ形成 |
